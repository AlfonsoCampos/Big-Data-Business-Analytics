Características

Es una infraestructura de almacén de datos que proporciona métodos de agregación, queries ad-hoc y análisis de grandes datasets almacenados en Hadoop empleando un dialecto SQL llamado Hive Query Language (HQL).

Desventajas:

  • no permite realizar actualizaciones, inserciones o borrados a nivel de registro.
  • alta latencia al estar creado sobre Hadoop que está orientado a procesos batch o de lotes
  • no proporciona transacciones

Ventajas

  • Hive es más adecuado para aplicaciones de data warehouse, donde grandes bloques de datos son analizados para realizar informes.
  • Facilidad de aprendizaje al basarse en SQL, frente a otras herramientas y lenguajes asociados con Hadoop, aunque hay que tener en cuenta sus propias peculiaridades al ser un dialecto de SQL y no seguir el estándar ANSI SQL.
