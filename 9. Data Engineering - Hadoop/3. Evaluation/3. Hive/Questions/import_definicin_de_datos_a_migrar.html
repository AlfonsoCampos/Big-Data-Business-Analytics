Import: definición de datos a migrar

El segundo bloque de argumentos que vamos a revisar son los argumentos que definen el tipo de datos que vamos a migrar así como información sobre el destino en el que queremos volcar dichos datos. 

Para poder importar una tabla o vista, emplearemos el comando --table seguido del nombre de la tabla. Por ejemplo podemos indicar

  --table empleados

Por defecto, todas las columnas de la tabla serán seleccionadas para la importación y en el mismo orden en el que se encuentran en la tabla original. Se pueden filtrar las columnas que se están migrando y ordenarlas de otro modo empleando el comando --columns seguido del nombre de las columnas

--columns apellidos,nombre, id_empleado

También se pueden filtrar los datos que se están migrando empleando el argumento --where

--where id_empleado> 500

Otra alternativa al uso de los comandos --table, --columns y --where es emplear el argumento --query que permite indicar la consulta completa. Este argumento debe ir seguido del argumento --targetdir para indicar la ruta en la que deseamos volcar la información

$ sqoop import \ 
  --query 'SELECT apellidos,nombre, id_empleado FROM empleados WHERE id_empleado>500' \
  --target-dir /user/bigdata/empleados
