Compresión

Existen distintos tipos de formato de compresión permitidos por Hive. 

Algunos de ellos son: 

GZIP o BZip2: 

CREATE TABLE raw (line STRING)
ROW FORMAT DELIMITED FIELDS TERMINATED BY '\t' LINES TERMINATED BY '\n';

LOAD DATA LOCAL INPATH '/tmp/weblogs/20141220-access.log.gz' INTO TABLE raw;

El inconveniente de emplear este tipo de archivos es que Hadoop no puede distribuir este tipo de ficheros en bloques para aprovechar el mapeo en paralelo. La solución es almacenar los datos en otra tabla que sí tenga almacenamiento del tipo SEQUENCEFILE. 

CREATE TABLE raw (line STRING)
ROW FORMAT DELIMITED FIELDS TERMINATED BY '\t' LINES TERMINATED BY '\n';

CREATE TABLE raw_sequence (line STRING)
STORED AS SEQUENCEFILE;

LOAD DATA LOCAL INPATH '/tmp/weblogs/20141220-access.log.gz' INTO TABLE raw;

SET hive.exec.compress.output=true;
SET io.seqfile.compression.type=BLOCK; 
INSERT OVERWRITE TABLE raw_sequence SELECT * FROM raw;

LZO: nuevo formato de compresión que favorece la velocidad frente a la tasa de compresión
