Introducción a Sqoop

Apache Sqoop es una herramienta diseñada para transferir de un modo eficiente bloques de datos entre Hadoop  (tanto de orígenes de datos sin estructura como HDFS o semiestructurados como Cassandra o HBase) y bases de datos estructuradas, como por ejemplo las bases de datos relacionales. 

Fue liberada de la incubadora de proyectos de Apache project en Marzo de 2012 y ahora es uno de sus proyectos más importantes. 

Sqoop proporciona una CLI (command line interface) con una serie de comandos que permiten dicha transferencia de datos. Permite cargas de o desde una única tabla de una base de datos o desde una query SQL, así como cargas desde scripts que pueden ejecutarse siempre que se necesite actualizar la información de una base de datos, desde la última importación. 

Sqoop proporciona un conjunto de conectores libres de alto rendimiento que pueden ser parametrizados para crear conexiones externas específicas.  Para poder realizar las transferencias de datos desde los sistemas de bases de datos relacionales, Sqoop emplea JDBC para acceder a dichos sistemas. Para poder emplear otros conectores, sólo hay que dejar en la ruta 

$SQOOP_HOME/lib

el driver que se desea emplear. De este modo Sqoop es capaz de acceder a la base de datos para identificar el esquema de datos que desea transferir. 

El modo de transferencia de datos tanto en la importación como en la exportación es mediante MapReduce. En el proceso de importación, Sqoop genera una clase Java que encapsula una de las filas de la tabla importada. Se puede acceder a dicho código Java para desarrollar nuevas aplicaciones que empleen un código similar al proceso Map Reduce generado.
