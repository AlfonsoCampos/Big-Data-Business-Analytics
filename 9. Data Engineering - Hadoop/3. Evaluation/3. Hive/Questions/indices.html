Indices

El objetivo de Hive empleando índices es mejorar la velocidad de búsqueda en las columnas de una tabla. Sin el índice las queries del estilo where tab1.col1 = 10, cargarían la tabla entera o partición para después procesar las filas. Sin embargo, si hay un índice sólo la porción de archivos necesarios se cargará para ser procesada. 

 

La mejora de queries empleando índices tiene como contraprestación el coste en la creación del índice y el espacio en disco necesario para almacenarlo. 

Create Index

Sintaxis:  

CREATE INDEX index_name
ON TABLE base_table_name (col_name, ...)
AS index_type
[WITH DEFERRED REBUILD]
[IDXPROPERTIES (property_name=property_value, ...)]
[IN TABLE index_table_name]
[
[ ROW FORMAT ...] STORED AS ...
| STORED BY ...
]
[LOCATION hdfs_path]
[TBLPROPERTIES (...)]
[COMMENT "index comment"];

Ejemplo: 

CREATE TABLE employees (
name STRING,
salary FLOAT,
subordinates ARRAY<STRING>,
deductions MAP<STRING, FLOAT>,
address STRUCT<street:STRING, city:STRING, state:STRING, zip:INT>
)
PARTITIONED BY (country STRING, state STRING);

CREATE INDEX employees_index
ON TABLE employees (country)
AS 'org.apache.hadoop.hive.ql.index.compact.CompactIndexHandler'
WITH DEFERRED REBUILD
IDXPROPERTIES ('creator = 'me', 'created_at' = 'some_time')
IN TABLE employees_index_table
PARTITIONED BY (country, name)
COMMENT 'Employees indexed by country and name.'; 

Drop Index

Sintaxis:  DROP INDEX [IF EXISTS] index_name ON table_name;

Ejemplo: DROP INDEX IF EXISTS employees_index ON TABLE employees; 

Alter Index

 Sintaxis: ALTER INDEX index_name ON table_name [PARTITION partition_spec] REBUILD;
