Almacenamiento en ficheros

Existen distintos tipos de almacenamientos en ficheros permitidos por Hive: 

  • SEQUENCEFILE: almacena el fichero como una secuencia única de pares claves-valores en binario que se guardan en Hadoop. Es bueno para compartir ficheros en el ecosistema de Hadoop
  • TEXTFILE (por defecto): almacena cada registro como una línea de texto. Es muy útil para compartir ficheros con otras aplicaciones y editar ficheros manualmente. Es menos eficiente que los ficheros de texto en formato binario al no emplear ningún tipo de estructura. 
  • RCFILE: almacena las columnas de la tabla como registros en forma de columna. Permite buenas compresión
  • ORC (optimized row columnar file): nuevo formato optimizado que podrá sustituir a RCFILE
  • PARQUET https://cwiki.apache.org/confluence/display/Hive/Parquet / http://parquet.incubator.apache.org/ formato de almacenamiento en columna disponible para cualquier proyecto del ecosistema de Hadoop.
  • AVRO http://avro.apache.org/ : sistema de serialización de datos de Hadoop que pretende convertirse en un estándar. Se apoya en la definición de un esquema para almacenar los datos. 

Cuando se define la tabla, se puede indicar el tipo de formato de archivo que vamos a emplear. Para ello se emplea el comando STORED AS FILEFORMAT. Es muy importante saber el tipo de uso que va a tener nuestro fichero para saber en qué formato vamos a almacenarlo. 

CREATE TABLE ejemplo1

STORED AS SEQUENCEFILE

Mediante la instrucción INPUT/OUTPUT FORMAT se puede especificar otro formato específico

CREATE TABLE ejemplo2

STORED AS 

INPUTFORMAT org.apache.hadoop.mapred.SequenceFileInputFormat

OUTPUTFORMAT org.apache.hadoop.hive.contrib.fileformat.base64.Base64TextInputFormat
