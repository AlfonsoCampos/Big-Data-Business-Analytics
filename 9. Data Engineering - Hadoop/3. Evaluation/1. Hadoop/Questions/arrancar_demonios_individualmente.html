Arrancar demonios individualmente

En /home/ubuntu/hadoop/sbin se localizan todos los scripts de arranque de cualquier demonio de Hadoop.

A continuación se muestran algunos ejemplos de arranque de esos demonios, algunos de los cuales se pueden arrancar con los scripts que vistos anteriormente o de forma individual. Es muy importante tenerlo en cuenta para el arranque individual de dichos demonios en un cluster con múltiples máquinas dedicadas. 

Namenode

 sbin/hadoop-daemon.sh start namenode

 jps

Datanode

sbin/hadoop-daemon.sh start datanode

 jps
 

Para arrancar denominos de MapReduce se ejecuta lo siguiente, en función del demonio a lanzar:

Resource Manager

sbin/yarn-daemon.sh start resourcemanager

jps


Node Manager

sbin/yarn-daemon.sh start nodemanager

jps

Job History Server

sbin/mr-jobhistory-daemon.sh start historyserver

jps

Todos los comandos anteriores se pueden ejecutar sin indicar la ruta sbin ya que dicha carpeta se introdujo en la ruta del PATH al modificar el bashrc. Por ejemplo podemos parar este demonio ejecutando el comando:

mr-jobhistory-daemon.sh stop historyserver
